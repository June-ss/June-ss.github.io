之前部署blog的时候，出现了这样的错误：

在网上搜了一些解决方案，第一次的时候用的是下面的方法 👇

完成了之后成功部署了。

当时我并不知道core.autocrlf是干嘛的，只是搜出来很多人都在说这个方法我就照做了。后来我去搜了一下core.autoarlf到底是什么，网上的解答说这个主要是解决换行符的问题。
具体的可以看一下这个帖子：How line ending conversions work with git core.autocrlf between different operating systems或者git的官方文档。简单来说如果是做跨平台项目时可能会需要把这个设置为true，不过如果是单平台的话设置成false就可以了（不过我没太搞懂设置这个跟解决spawn failed有什么关系？）

然而上面的方法好像不能一劳永逸，后来我还是遇到了同样的错误。这一次我尝试了第二种方法，把_config.yml文件里的repo修改成ssh格式：
👉git@github.com:<username>/<username>.github.io.git
比如我的就修改成@github.com:June-ss/june-ss.github.io.git，and it worked:)
可能因为我之前设置了ssh key吧😀（这样每一次push的时候都可以不用密码）
PS 关于ssh协议，可以参考一下这篇文章：What is the SSH Protocol，关于SSH和HTTPS的对比，可以参考一下这个帖子：What is the difference between SSH and HTTPS?

还有，如果修改配置文件的话，记得注意一下拼写。比如我在修改repo格式的时候就写错了一个标点符号，结果一直出现spawn failed，试遍了网上的方法，又去看了一遍hexo的官方文档，还是无法解决问题，直接把我给整不会了——直到我回去看了一眼配置文件:)
