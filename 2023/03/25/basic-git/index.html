什么是Git

一个分布式的版本控制工具。

  • 工作流程：
    在工作目录修改文件 → 在暂存区创建快照 → 提交到Git仓库
    ie. working directory → staging area → .git directory (repository)
  • Git的三种状态：已修改(modified)、已暂存(staged)、已提交(committed)
    已修改：在本地修改了文件，但没有保存到数据库中
    已暂存：标记已修改的文件的当前版本（将更改的文件添加到暂存区中，创建快照）
    已提交：将暂存区中的快照提交（保存）到Git仓库中

    初始化之后，当前项目目录下的文件会有两种状态，一种是已跟踪（tracked），一种是未跟踪（untracked）。已跟踪的文件指已经被纳入Git的文件（Git已经知道的文件）。除了已跟踪的文件之外，就是未跟踪的文件，既不在上次快照的记录中，也没有被放到暂存区中。

    • 当你对本地的文件做出了修改时，使用add命令来将文件添加到暂存区：

      注：提交到仓库之后，暂存区中的文件不会自动删除。如果工作目录中有文件1和文件2，然后你删除了文件2，而文件2已经被Git跟踪存放在暂存区中，那么应该使用git add file2指令来将文件2从暂存区中删除。

    • 查看当前仓库状态：

      在简览状态中，新添加的未跟踪文件前面有??标记，新添加到暂存区中的文件前面有A标记，修改过的文件前面有M标记。输出中有两栏，左栏指明了暂存区的状态，右栏指明了工作区的状态。

    • 当你确定好所有的修改准备就绪时，使用commit命令来将文件提交到Git仓库：

      注释（message）：简单地描述你的这次提交，比如fix bugs, implement new feature, refactor code等等，方便查看当前提交进行了哪些修改。

      PS：
      1) not too big and not too small
      2) commit often
      3) as you reach a state you want to record, then make a commit
      4) do not mix things (eg. seperate the bug-fixed and typo, make two commits)
      5) create a meaningful commit message
